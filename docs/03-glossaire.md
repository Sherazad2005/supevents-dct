# Glossaire

| Terme | Définition | Première occurrence |
|---|---|---|
| SSO | Single Sign-On — mécanisme permettant aux étudiants et organisateurs de se connecter une seule fois pour accéder à SupEvents et aux services associés. | §4 |
| OIDC | OpenID Connect — protocole d’authentification utilisé avec le SSO pour sécuriser la connexion des utilisateurs sur SupEvents. | §4 |
| RBAC | Role-Based Access Control — gestion des permissions selon le rôle utilisateur (étudiant, organisateur, administrateur). | §4 |
| RGPD | Règlement Général sur la Protection des Données — réglementation imposant la protection des données personnelles des utilisateurs de SupEvents. | §4 |
| PCI-DSS | Norme de sécurité obligatoire pour le traitement des paiements en ligne sur SupEvents. | §4 |
| e-ticket | Billet numérique généré après l’inscription à un événement étudiant sur SupEvents. | §5 |
| CRUD | Create, Read, Update, Delete — opérations de base réalisées sur les événements, utilisateurs et réservations. | §4 |
| SLA | Service Level Agreement — engagement de disponibilité et de performance du système SupEvents. | §4 |
| webhook | Mécanisme permettant à SupEvents d’envoyer automatiquement des notifications à des services externes. | §4 |
| broker de messages | Service intermédiaire gérant les échanges asynchrones entre plusieurs composants du système. | §4 |
| API REST | Interface de communication HTTP utilisée entre le front-end et le back-end de SupEvents. | §4 |
| P95 | Indicateur de performance mesurant le temps de réponse maximal pour 95 % des requêtes utilisateur. | §4 |
| WCAG | Web Content Accessibility Guidelines — normes garantissant l’accessibilité du site aux personnes en situation de handicap. | §4 |
| i18n | Internationalisation — capacité de SupEvents à supporter plusieurs langues et formats régionaux. | §4 |
| CI/CD | Continuous Integration / Continuous Deployment — automatisation des tests et du déploiement du projet SupEvents. | §4 |
